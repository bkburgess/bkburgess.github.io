Bayesian Optimization Overview

Bayesian Optimization is a search algorithm for finding the best place to sample for an expensive to calculate function. One of the most common applications for this method is in tuning the hyperparameters of machine learning models such as neural networks. It usually uses a small sample of points to shepherd a collection tens of distinct smooth curves. Then for each point in the domain, you calculate the variance in the values given by the curves. Because the curves converge at known points, and tend to spread out in between known points, the algorithm can balance prioritizing its search between exploiting the best values found so far, against the value of exploring new areas to gather more information. Additionally, this balance can be tuned with a single input parameter.

Exploratory Optimization

The exploratory_opt.py file was inspired by bayesian optimization, but instead of calculating the spread of a collection of smooth function, it just fits any one of several convex functions to pass through each pair of adjacent sampled points. The resulting piecewise function can be used to achieve a similar overall result that balances exploiting the best samples found so far against exploring unknown areas, again tunable with a single parameter. This implementation uses a standin function for the expensive function that it's supposed to be applied to, by composing several sine waves of different amplitudes and frequencies.

PID Overview

PID controllers are widely used in robotics, where they can determine how to get the robot, or a part of a robot to the desired state quickly, efficiently, and stably, even in the face of interference and outside forces. The controller determines how much force to apply in which direction. To do this they consist of essentially three parts:
Proportional - this just measures the difference between where the robot is, and where it's trying to get to, and provides a force proportional to that distance. If this were the only component, then the robot would oscillate around the target point with amplitude equal to the initial difference, rather than stay consistently at or near the target.
Derivative - this measures how much progress the robot has made in each computation cycle, which helps the robot avoid overshooting or oscillating around the target, by providing a slight damping effect proportional to the progress it's already made.
Integrative - this measures how much progress the robot has gone, and is primarily useful for compensating for outside effects like external forces, or changes in the robots mass, which affect the relationship between the robot's force applied and the motion that results. This can sometimes be left out in simpler application
Each of these components is multiplied by a tuned gain value to determine the influence that they each have on the force applied.

SIRD

This repository is a place for me to work on epidemiology simulations, primarily centered on using the SIRD model. SIRD stands for Susceptible, Infected, Recovered, and Dead. The idea is that the population of a a densely populated area will fall into one of these four categories at any given time, and we can use estimates and heuristics to model how the disease progresses over time. Patient Choice

Skills/Technologies: Python, Pandas

This is a project I worked on at TU Berlin, to help the German legislature decide how they might be able to streamline their healthcare system. I was provided data about individual hospitals, demographic data, and patient healthcare data sourced from health insurance companies in Germany with the goal of estimating what factors most influenced a patients' choice of hospital for elective treatment, to aid in validating a colleague's assumption. In particular, I wanted to know how much impact the quality of the hospital had on patients' decision, since Germany has a national and publicly available (by website) system for evaluating hospitals based on their quality. Price and insurance network were nonfactors because of the nature of Germany's healthcare system.

Ultimately, I found that the single most impactful factor was simply the travel time between the patients' home location and the hospital.

Note that because I was working with very sensitive personal health information, I am not providing any of the data I was using, nor anything that could expose that information.

Dragon Curve Fractal

Skills/Technologies: Python

This script generates the dragon fractal, which is the logical extension of folding a piece of paper in half over and over, and then unfurling it so that each of the bends is 90 degrees. To do this, this script encodes directions as a string of numbers, and recurrently duplicates and rotates the string to form a new string which represents the next iteration of the fractal. It finally then needs to decode the resulting string back into directions that can be used to plot the fractal itself.
